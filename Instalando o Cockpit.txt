                Para instalar o Cockpit no servidor: DEBIAN 12.7.0

    1. sudo apt -y install cockpit <= Este comando instala o Cockpit no sistema.
    2. cd /lib/systemd/system <= Este comando navega até o diretório onde os arquivos de configuração de serviços do systemd estão localizados.
    3. sudo vi cockpit.socket <= Este comando abre o arquivo de configuração do cockpit.socket no editor de texto vi, porém faça um backup antes de editar, ex: sudo cp cockpit.socket cockpit.socket.bkp
    4. Na linha 07 = ListenStream=9090 <= Aqui, você está modificando a configuração do serviço Cockpit para definir a porta em que o serviço será executado.
    • Explicação: Dentro do arquivo cockpit.socket, a linha ListenStream=9090 define a porta de rede que o serviço Cockpit usará para escutar conexões. O valor padrão pode ser 9090 ou 9091, dependendo da configuração padrão do sistema, mas você pode querer mudá-la se houver um conflito de portas ou se desejar uma configuração personalizada. O número 9090 é uma escolha comum, pois é a porta padrão usada pelo Cockpit. Ao editar esse arquivo, você pode personalizar o número da porta conforme necessário.
    5. sudo systemctl daemon-reload <= Este comando recarrega as configurações do systemd.
    6. sudo systemctl restart cockpit.socket <= Este comando reinicia o serviço cockpit.socket para aplicar as mudanças feitas no arquivo de configuração.
    • Explicação: Após modificar a configuração da porta no arquivo cockpit.socket, você precisa reiniciar o serviço para que as mudanças tenham efeito. O systemctl restart reinicia o serviço cockpit.socket e faz com que ele comece a escutar na nova porta definida (neste caso, na porta 9090).

    • Pronto seu cockpit, já está funcionando.

Comando:

    1. sudo systemctl enable cockpit.socket <= Este comando habilita o serviço cockpit.socket para que ele seja iniciado automaticamente na inicialização do sistema. 
    • Explicação: O systemctl enable cria links simbólicos para o serviço no diretório /etc/systemd/system/ que indicam que o serviço deve ser iniciado automaticamente quando o sistema for iniciado.
    2. sudo systemctl stop cockpit.socket  <= Este comando interrompe imediatamente o serviço cockpit.socket.
    • Explicação: O systemctl stop é usado para parar um serviço em execução no sistema, neste caso, o cockpit.socket, o que significa que a interface gráfica do Cockpit não estará mais acessível até que o serviço seja reiniciado.
    3. sudo systemctl restart cockpit.socket  <= Este comando reinicia o serviço cockpit.socket.
    • Explicação:  O comando reinicia o serviço de rede (socket) que permite ao Cockpit funcionar corretamente, especialmente quando você acessar o painel web do Cockpit para gerenciar e monitorar o sistema.
    4. sudo systemctl start cockpit.socket  <= Este comando inicia o serviço cockpit.socket.
    • Explicação: O systemctl start é utilizado para iniciar um serviço que não está em execução. Neste caso, ele iniciaria o cockpit.socket, permitindo que o Cockpit seja acessado pela interface web.

    • https://IP_DO_SERVIDOR:9090/ <= refere-se ao endereço URL para acessar a interface web do Cockpit, que é uma ferramenta de administração para sistemas Linux. 

    • Observação: Esses comandos são essenciais para gerenciar o funcionamento do Cockpit de forma eficiente e para garantir que ele esteja disponível sempre que necessário, seja manualmente iniciado ou automaticamente iniciado na inicialização do sistema.
